# Phase 1 PRD：录音转写 + AI 会议总结

**版本：** Phase 1 v2.0
**状态：** ✅ 已完成
**交付目标日期：** Phase 0 完成后 +2 周

## 1. 背景与目标

### 1.1 背景

企业客户（AI 耳机厂商、会议设备厂商）普遍面临会议效率低下的痛点：
- 会议录音无人整理，关键决策遗漏
- 人工转写成本高、耗时长（1小时会议需 3 小时整理）
- 多人发言无法区分，追责困难

本阶段通过 Azure Speech 转写 + GPT-4o 总结，**30 秒内完成 1 小时会议纪要**，直观展示 Azure AI 的处理能力。

### 1.2 目标

| 目标类型 | 描述 |
|----------|------|
| 演示目标 | 让客户亲眼看到：上传音频 → 自动转写 → AI 生成结构化纪要 全流程 |
| 技术目标 | 验证 Azure Speech Transcription + GPT-4o 完整链路可用性 |
| 体验目标 | 操作步骤 ≤ 3 步，非技术人员可独立操作 |

| 用户 | 场景 | 核心诉求 |

## 6. 技术方案

### 6.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| UI 框架 | **Flet 0.28+** | Material Design 3，暗色主题，圆角卡片（实测 0.28.3） |
| 语音转写 | `azure-cognitiveservices-speech` ≥1.37.0 | 官方 Speech SDK，`ConversationTranscriber` 支持 Diarization |
| AI 总结 | `openai` Python SDK ≥1.30.0 | 调用 Azure OpenAI GPT-4o，Streaming |
| 音频录制 | `sounddevice` + `scipy` | 麦克风采集，保存为 .wav |
| 打包 | `PyInstaller` | 生成单一 .exe，客户双击运行 |

### 6.2 Azure 资源依赖

| 资源 | 用途 | 备注 |
|------|------|------|
| Azure AI Speech | 音频转写 + Diarization | 推荐 `eastasia` 区域 |
| Azure OpenAI | GPT-4o 生成纪要（Streaming） | 部署 `gpt-4o` 模型 |

### 6.3 核心调用链路

```
音频文件 / 麦克风录音（.wav）
    ↓
azure-cognitiveservices-speech SDK ConversationTranscriber
→ 带 Speaker 标注 + 时间戳的转写结果
    ↓
Flet UI 渲染：按 Speaker 分色气泡卡片（ft.Card）
    ↓
用户点击「生成纪要」
    ↓
Azure OpenAI GPT-4o Chat Completion（stream=True）
    ↓
Flet ft.Markdown 实时流式渲染纪要内容
```

# Phase 2 PRD：GPT-4o Realtime 实时语音对话

**版本：** Phase 2 v2.0
**状态：** ✅ 已完成
**交付目标日期：** Phase 1 完成后 +2 周

---

## 1. 背景与目标

### 1.1 背景

AI 玩具、AI 耳机的核心交互体验是**自然语音对话**。传统方案需要 ASR → LLM → TTS 三段拼接，端到端延迟普遍超过 2 秒，用户体验割裂。

Azure OpenAI GPT-4o Realtime API 实现**音频直接输入、音频直接输出**，首包延迟可低至 **300ms 以内**，这是本阶段最核心的演示卖点。

### 1.2 目标

| 目标类型 | 描述 |
|----------|------|
| 演示目标 | 客户亲身体验：说话 → AI 毫秒级回应，全程无文字中转 |
| 技术目标 | 验证 GPT-4o Realtime WebSocket 链路延迟 |
| 对比目标 | 界面直观展示「传统三段式 vs Realtime」实测延迟对比图表 |

---

## 2. 目标用户

| 用户 | 场景 | 核心诉求 |
|------|------|----------|
| AI 玩具厂商技术负责人 | 验证儿童语音交互方案 | 延迟低、内容安全、声音自然 |
| AI 耳机产品经理 | 验证 AI 助手耳机方案 | 流畅度、打断处理 |
| Azure 售前工程师 | 演示 Azure OpenAI 旗舰能力 | 震撼演示效果、稳定不出错 |

```

response.audio_transcript.delta → Flet 气泡卡片实时更新
    ↓
response.done → 性能指标（TTFT/E2E/Token）更新
```

# Phase 3 PRD：同声传译（Live Interpreter）

**版本：** Phase 3 v2.0
**状态：** ✅ 已完成
**交付目标日期：** Phase 2 完成后 +2 周

---

## 1. 背景与目标

### 1.1 背景

AI 耳机最强力的应用场景是**跨语言实时翻译**：佩戴耳机参加国际会议，对方说英文，耳机里实时传来中文翻译。Azure Live Interpreter API 集中展示了 Azure 的核心优势：
- 语种**自动检测**，无需手动设置
- 实时流式语音翻译，延迟 < 500ms
- 高质量神经网络 TTS 输出，音色自然

### 1.2 目标

| 目标类型 | 描述 |
|----------|------|
| 演示目标 | 说中文，实时出现英文字幕+语音；说英文，实时出现中文字幕+语音 |
| 技术目标 | 验证 Azure Speech Translation Live Interpreter API 端到端链路 |
| 差异化目标 | 展示「自动语言检测」，无需用户手动切换，突出 AI 智能化 |

| 用户 | 场景 | 核心诉求 |

| 国际化企业采购方 | 跨国会议翻译工具评估 | 准确率高、多语言支持 |
